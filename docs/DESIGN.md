# DESIGN — 디자인 가이드

## 1. 디자인 방향

**미니멀·프리미엄** 톤.
핵심 원칙: 큰 사진이 주인공, 넉넉한 여백, 절제된 텍스트, 흑·백·회 중심의 차분한 팔레트.
"꾸미기"보다 "비우기". 사진이 돋보이도록 UI는 최대한 조용하게.

## 2. 컬러 팔레트

화이트 기반 + 차콜/그레이 텍스트 + 브랜드 티알 포인트.

```
배경(기본)      #FFFFFF   (brand-white)
배경(보조)      #F7F6F4   (brand-offwhite, 섹션 구분용)
텍스트(강)      #1A1A1A   (brand-dark, 제목)
텍스트(본문)    #3D3D3D   (brand-body)
텍스트(보조)    #8A8A8A   (brand-muted, 캡션·메타)
보더/구분선     #E5E3DF   (brand-border)
브랜드 포인트   #00667F   (brand-teal — CTA 버튼·액센트·호버 오버레이)
```

> 모든 색상 토큰은 `tailwind.config.ts`의 `theme.extend.colors.brand`에 등록되어 있음.

## 3. 타이포그래피

- 한글 본문: Pretendard (권장) 또는 시스템 산세리프
- 영문/슬로건: 깔끔한 산세리프(Inter 등). 슬로건은 자간 넓게(tracking-wide), 대문자
- 위계
  - Hero 슬로건: 매우 큼, 굵게, 자간 넓게
  - 섹션 타이틀: 28~40px, 중간 굵기
  - 본문: 15~17px, line-height 넉넉히(1.7~1.9)
  - 캡션: 13~14px, 보조 색상
- 문단 사이 여백 충분히. 텍스트 블록은 가운데 정렬 또는 좌측 정렬 일관 유지

## 4. 레이아웃 · 간격

- 최대 콘텐츠 폭: 약 1200~1280px, 가운데 정렬
- 좌우 패딩: 모바일 20px / 태블릿 40px / PC 60~80px
- 섹션 상하 여백: 모바일 64px / PC 120px 이상 (여백 넉넉하게)
- 그리드 갭: 16~24px

## 5. 반응형 브레이크포인트 (Tailwind 기본)

```
기본(모바일)   ~ 767px     1열, 햄버거 메뉴
md (태블릿)    768px ~     2열
lg (PC)        1024px ~    3열, 가로 네비
xl             1280px ~    최대폭 적용
```

mobile-first 작성. 모든 그리드 기본 1열 → md 2열 → lg 3열.

## 6. 컴포넌트 스타일

### 헤더/네비

- 투명 또는 화이트 배경, 상단 고정(sticky), 스크롤 시 약한 그림자/보더
- 로고 좌측, 메뉴 우측, [상담 문의]는 아웃라인 또는 솔리드 버튼
- 포트폴리오: hover/click 드롭다운(데스크톱), 아코디언(모바일)

### 히어로

- 풀스크린(100vh 또는 큰 높이) 배경 이미지, 어두운 오버레이 약간
- 중앙 슬로건 + 서브카피 + CTA 버튼
- 스크롤 다운 인디케이터(선택)

### 카드 (서비스/포트폴리오)

- 이미지 + 제목 + 짧은 설명. 보더 최소, 그림자 거의 없음
- hover 시 이미지 살짝 확대(scale) 또는 오버레이 — 부드러운 transition

### 버튼

- 솔리드(주요 CTA): 다크 배경 + 화이트 텍스트
- 아웃라인(보조): 얇은 보더
- 라운드는 작게(rounded-sm~md) 또는 각지게. 과한 둥글기 지양

### 푸터

- 오프화이트/다크 배경. 로고, 소셜 아이콘, 영업시간, 지점 주소, 사업자 정보
- 작은 텍스트, 보조 색상

## 7. 이미지

- 모두 `next/image`, lazy load
- 갤러리/카드: object-cover로 비율 통일(예: 4:3 또는 3:2)
- 히어로: 고해상도, 모바일/데스크톱 각각 적절한 사이즈
- 실제 이미지 들어오기 전: `/public/images/placeholder/` 더미 사용

## 8. 모션

- 절제된 모션만. 스크롤 시 페이드인/업(은은하게), hover transition 200~300ms
- 과한 애니메이션 금지 — 프리미엄 톤 유지
